---
title: "简介"
description: "了解 Arupa 内核及其如何托管插件。"
---

Arupa 内核是 Arupa 的核心宿主进程。它提供插件赖以构建的共享服务器运行时环境，因此每个插件都可以专注于自身的功能，而无需自行实现服务器、身份验证或插件间的通信。

内核负责：

- 处理 HTTP 请求和 Socket.IO 连接；

- 发现、加载、启动和停止插件，以及重新加载配置；

- 将插件声明的 HTTP 路由、静态文件和 Socket.IO 命名空间连接到服务器；

- 提供共享主机 API，例如键值存储、日志记录、事件发射和插件消息；

- 对请求进行身份验证，并强制执行已配置的主机、插件、路由和事件访问策略。

插件作为独立的 WASM 或 gRPC 组件运行。注册期间，插件

会声明它提供的路由、静态挂载点和 Socket.IO 命名空间。

内核随后会将匹配的请求和事件分派给所属插件，同时
将主机级别的身份和访问检查集中在一个地方。

要运行 Arupa，首先需要安装内核，然后选择适合您环境的安装方法。
