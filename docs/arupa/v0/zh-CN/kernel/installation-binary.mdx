---

title: "从二进制文件安装"

description: "在 Linux 或 macOS 上从已发布的二进制文件安装并启动 Arupa 内核。"

---

使用此方法可直接在 Linux 或 macOS 主机上运行 Arupa 内核。

该版本包含一个可执行文件。它不包含配置文件或插件，因此您需要在以下步骤中创建内核目录及其初始文件。

## 开始之前

已发布的二进制文件适用于 Linux 和 macOS，支持 `amd64` 和 `arm64` 处理器。Windows 目前不是二进制文件发布目标平台。

下载版本之前，请查找您的操作系统和处理器架构：

```sh
uname -s
uname -m
```

选择匹配的版本：

| `uname -s` | `uname -m` | 版本后缀 |
| --- | --- | --- |
| `Darwin` | `arm64` | `darwin-arm64` |
| `Darwin` | `x86_64` | `darwin-amd64` |
| `Linux` | `aarch64` | `linux-arm64` |
| `Linux` | `x86_64` | `linux-amd64` |

## 1. 下载内核

打开 [Arupa 发布页面](https://github.com/SteelDrEgg/Arupa/releases)。选择一个版本，然后下载后缀与您的系统匹配的资源。

发布文件使用以下格式：

```text
arupa-v<版本>-<操作系统>-<架构>
```

例如，运行 `v0.2.0` 版本的 Apple Silicon Mac 使用以下格式：

```text
arupa-v0.2.0-darwin-arm64
```

## 2. 创建内核目录

将可执行文件、配置文件、插件包和临时解压文件放在同一个目录中。以下命令会在您的主目录中创建一个名为 `arupa` 的目录，并将示例下载文件移动到该目录中：

```sh
mkdir -p ~/arupa
mv ~/Downloads/arupa-v0.2.0-darwin-arm64 ~/arupa/arupa
cd ~/arupa
```

如果您下载了不同的发布文件，只需在 `mv` 命令中替换源文件名即可。最终名称 `arupa` 就是您将要运行的可执行文件。

## 3. 允许二进制文件运行

将下载的文件设置为可执行文件，然后确认其可以运行：

```sh
chmod +x arupa
./arupa -version
```

该命令会打印已安装的内核版本。如果 macOS 阻止二进制文件打开，请查看安全提示，
并在确认您是从官方发布页面下载后，在 **系统设置** 中允许该应用程序。

## 4. 创建初始配置

内核会读取一个 TOML 配置文件。创建两个用于默认插件设置的两个目录：

```sh
mkdir plugins
mkdir tmp
```

在 `~/arupa` 目录下创建一个名为 `config.toml` 的文件，并添加以下内容：

```toml
Listen = ":8080"
PluginDir = "plugins"
PluginTempDir = "tmp"
```

`Listen` 是内核接受 HTTP 和 Socket.IO 连接的地址和端口。

`PluginDir` 存放已下载的 `.plg` 插件包。

内核在加载这些包时使用 `PluginTempDir`。这些相对路径是从 `~/arupa` 解析的，因为这是运行内核的目录。

## 5. 启动并验证内核

从 `~/arupa` 启动内核：

```sh
./arupa -config config.toml
```

保持此终端运行。在另一个终端中，验证内核是否响应：

```sh
curl http://127.0.0.1:8080/api/kernel/version
```

响应中包含内核版本。在第一个终端中按 `Ctrl-C` 停止内核。

## 后续步骤

内核现在可以安装插件包了。请继续阅读 [Coreplugins文档](../../../../coreplugins/v0/en/index)，
然后按照 [安装插件](./plugins#install-a-plugin) 的步骤将插件包添加到 `plugins`目录中。
在将内核暴露于本地计算机之外之前，请配置[用户](./config-users)和[访问规则](./config-access)。
