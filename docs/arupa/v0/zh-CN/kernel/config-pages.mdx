`Pages` 部分将 HTTP 状态码与 URL 路径关联起来：

```toml
[Pages]
401 = "/pages/login.html"
```

内核在需要决定是否将浏览器请求重定向到另一个页面时，
会查找状态码对应的路径。该值必须是以 `/` 开头的本地 URL 路径。
绝对 URL（例如`https://example.com/login`）和协议相对 URL（例如`//example.com/login`）将被拒绝。

## 浏览器请求

内核仅对浏览器风格的 `GET` 请求使用已配置的页面。

请求的目标路径不得为 `/api` 或 `/api` 下的路径，
并且其 `Accept` 标头必须为空或包含 `text/html`。

当这些条件不满足时，请求将保持正常的 API 或 HTTP 响应，而不是被重定向。

如果请求已经配置了页面路径，内核也会保持其不变，以避免页面重定向到自身。重定向使用 HTTP `303 See Other` 状态码。

## 用途

映射充当请求流的导航机制。

它处理诸如以下行为：当请求受保护的路由时，将未经身份验证的用户重定向到登录页面；

或者当插件路由器找不到匹配的路由或静态挂载点时，显示 `404` 页面。
